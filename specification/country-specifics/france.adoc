This section contains information about requirements and other information concerning invoicing in France.

==== French E-Invoicing Mandate (2026)

France implemented a mandatory B2B/B2G e-invoicing system starting in 2026, requiring all invoices between French entities to be transmitted through certified platforms (Plateformes Agréées - PA).

==== Invoice Type Code and Business Process Type

In addition to the standard invoice type code, France requires the business process type to be specified using the `@name` attribute on `cbc:InvoiceTypeCode`.

The business process type (Cadre de facturation) indicates the nature of the transaction and affects VAT treatment. For the complete list of supported codes and their descriptions, see the https://pagero.github.io/puf-code-lists/#_business_process_codes_for_france[PUF Code Lists specification^].

*Example for Goods Invoice*
[source,xml]
----
<Invoice>
    <cbc:InvoiceTypeCode name="B1">380</cbc:InvoiceTypeCode>
    <!-- Code omitted for clarity -->
</Invoice>
----

==== French Text Codes (Note Elements)

France requires specific information to be provided in invoice notes using standardized text code prefixes. These text codes identify the type of legal, regulatory, or commercial information being provided.

===== Text Code Format

Each text code must be provided in a separate `cbc:Note` element with the format: `#CODE#value`

===== Available Text Codes

[cols="1,2,3,2", options="header"]
|===
|Code |Name |Description |Required Value (if applicable)

|#ABL#
|Legal information
|Trade register number (RCS) or other legal registration information
|

|#PMD#
|Late payment penalties
|Payment terms regarding late payment penalties
|

|#PMT#
|Collection costs (€40)
|Fixed €40 indemnity for collection costs in case of late payment
|

|#AAI#
|General information
|General elements typically found at the bottom of paper invoices (e.g., APE code, capital social)
|

|#BAR#
|Treatment type qualification
|Expected treatment type for the invoice
|*One of:* `B2B`, `B2BINT`, `B2C`, `OUTOFSCOPE`, `ARCHIVEONLY`

|#TXD#
|Single taxable person member
|Indication that member is part of a single taxable person (for transactions external to the group)
|*Required:* `Membre d'un assujetti unique`

|#BLU#
|Eco-participation/Eco-contribution
|Eco-participation or Eco-contribution WEEE (can also be used for other taxes including eco-tax)
|*Required:* `Eco-participation (L. 541-10 du code de l'environnement)` or `Eco-contribution DEEE`

|#AAB#
|Early payment discount
|Statement whether customer can apply discount if paying before due date
|

|#ACC#
|Subrogation factoring clause
|Factoring arrangement details and subrogation clauses
|

|#SUR#
|Supplier notes
|Additional remarks from the supplier
|

|#CUS#
|Customs information
|Customs-related details and information
|

|#PAI#
|Third party payment
|Information about third-party payment arrangements
|

|#ADN#
|B2G indicator
|Indicates if invoice is subject to B2G treatment (additional CHORUS PRO rules apply)
|
|===

IMPORTANT: 

* The #PMT# fixed indemnity amount is legally set at €40.00 per Article L441-10 of the French Commercial Code
* #BAR# is mandatory and must contain exactly ONE of the specified values: `B2B` (e-invoicing), `B2BINT` (B2B international e-reporting), `B2C` (B2C e-reporting), `OUTOFSCOPE` (outside reform scope), or `ARCHIVEONLY` (internal credit note - no transmission)
* Only ONE #BAR# note is allowed per invoice

==== Self-Billing Indicator

Self-billing invoices can be indicated in two ways:

1. **Using Invoice Type Code**: Set `cbc:InvoiceTypeCode` to `389` (self-billed invoice) or `261` (self-billed credit note) with the appropriate https://pagero.github.io/puf-code-lists/#_business_process_codes_for_france[business process code^] in the `@name` attribute
2. **Using Extension Indicator**: Use the dedicated `SelfBilled` extension with value `true`

Both methods are supported, and implementations may choose either approach based on their requirements.

*Example using Invoice Type Code*
[source,xml]
----
<Invoice>
    <!-- Code omitted for clarity -->
    <cbc:InvoiceTypeCode name="B1">389</cbc:InvoiceTypeCode>
    <!-- Code omitted for clarity -->
</Invoice>
----

*Example using Extension Indicator*
[source,xml]
----
<Invoice>
    <!-- Code omitted for clarity -->
    <ext:UBLExtensions>
        <ext:UBLExtension>
            <ext:ExtensionURI>urn:pagero:ExtensionComponent:1.0:PageroExtension:SelfBilled</ext:ExtensionURI>
            <ext:ExtensionContent>
                <puf:PageroExtension>
                    <puf:SelfBilled>true</puf:SelfBilled>
                </puf:PageroExtension>
            </ext:ExtensionContent>
        </ext:UBLExtension>
    </ext:UBLExtensions>
    <!-- Code omitted for clarity -->
</Invoice>
----

==== Business and VAT Identification

France uses SIRET (14-digit) and SIREN (9-digit) numbers for business identification.

===== Endpoint Identification

For electronic addressing in the French e-invoicing system, **schemeID 0225** is the primary electronic addressing scheme. The endpoint value can contain composite information combining SIREN, SIRET, and optional routing identifiers.

**Endpoint Registration Formats (schemeID 0225):**

There are four possible registration formats in the French e-invoicing directory (annuaire):

1. `SIREN` - Company identifier only
2. `SIREN_SUFFIX` - Company identifier with custom suffix
3. `SIREN_SIRET` - Company and establishment identifiers
4. `SIREN_SIRET_CODEROUTAGE` - Full identifier with routing code

Where:

* **SIREN** (schemeID 0002) - 9-digit company identifier
* **SIRET** (schemeID 0009) - 14-digit establishment identifier
* **SUFFIX** - Custom suffix appended directly to SIREN (no separate scheme code)
* **CODE ROUTAGE** (schemeID 0224) - Buyer-defined routing code for internal distribution (e.g., department, cost center, or business unit)

NOTE: SUFFIX and CODE ROUTAGE achieve similar functionality at different organizational levels. Both are entirely determined by the buyer for routing invoices to the appropriate internal recipient. The platform must register such addresses in the French e-invoicing registry.

**Party Structure Requirements:**

In addition to the endpoint (schemeID 0225), the individual identifier components should be provided in the party structure:

* **SIREN** → `PartyLegalEntity/CompanyID` with schemeID `0002`
* **SIRET** → `PartyIdentification/ID` with schemeID `0009`
* **CODE ROUTAGE** → `PartyIdentification/ID` with schemeID `0224` (when applicable)

*Example with Electronic Address (schemeID 0225)*
[source,xml]
----
<Invoice>
    <!-- Code omitted for clarity -->
    <cac:AccountingSupplierParty>
        <cac:Party>
            <!-- Endpoint: SIREN -->
            <cbc:EndpointID schemeID="0225">123456789</cbc:EndpointID>
            <cac:PartyIdentification>
                <cbc:ID schemeID="0009">12345678901234</cbc:ID>
            </cac:PartyIdentification>
            <cac:PartyLegalEntity>
                <cbc:CompanyID schemeID="0002">123456789</cbc:CompanyID>
                <cbc:RegistrationName>Supplier Company Name</cbc:RegistrationName>
            </cac:PartyLegalEntity>
            <!-- Code omitted for clarity -->
        </cac:Party>
    </cac:AccountingSupplierParty>
    <cac:AccountingCustomerParty>
        <cac:Party>
            <!-- Endpoint: SIREN_SIRET_CODEROUTAGE -->
            <cbc:EndpointID schemeID="0225">987654321_98765432109876_DEPT01</cbc:EndpointID><!--1-->
            <cac:PartyIdentification>
                <cbc:ID schemeID="0009">98765432109876</cbc:ID><!--2-->
            </cac:PartyIdentification>
            <cac:PartyIdentification>
                <cbc:ID schemeID="0224">DEPT01</cbc:ID><!--3-->
            </cac:PartyIdentification>
            <cac:PartyLegalEntity>
                <cbc:CompanyID schemeID="0002">987654321</cbc:CompanyID><!--4-->
                <cbc:RegistrationName>Buyer Company Name</cbc:RegistrationName>
            </cac:PartyLegalEntity>
            <!-- Code omitted for clarity -->
        </cac:Party>

==== Service code

Service code is a routing code required by some customers to identify the correct recipient in French Government Portal.

It's used as an addition to the https://en.m.wikipedia.org/wiki/SIRET_code[SIRET number^], which is the main identifier for all government recipients.

The Service code should be provided in the extension block under <<_cacaccountingcustomerparty, cac:AccountingCustomerParty/cac:Party>>, in the structure `puf:PageroExtension/puf:PartyExtension/puf:ADAID`.

More information how to use ADAID can be found <<_additional_destination_address_identifier_adaid, here>>.

See example below, note that `cbc:Name` is not mandatory to send, but opens up the possibility to provide the name of the receiving unit.

*Example*
[source,xml]
----
<Invoice>
    <!-- Code omitted for clarity -->
    <cac:AccountingCustomerParty>
        <!-- Code omitted for clarity -->
        <cac:Party>
            <ext:UBLExtensions>
                <ext:UBLExtension>
                    <ext:ExtensionURI>urn:pagero:ExtensionComponent:1.0:PageroExtension:PartyExtension</ext:ExtensionURI>
                    <ext:ExtensionContent>
                        <puf:PageroExtension>
                            <puf:PartyExtension>
                                <puf:ADAID>
                                    <cbc:ID>1234567890</cbc:ID>
                                    <cbc:Name>Name of receiving unit</cbc:Name>
                                    <puf:IDType listID="PUF-002-ADAID">FR:ServiceCode</puf:IDType>
                                </puf:ADAID>
                            </puf:PartyExtension>
                        </puf:PageroExtension>
                    </ext:ExtensionContent>
                </ext:UBLExtension>
            </ext:UBLExtensions>
            <!-- Code omitted for clarity -->
        </cac:Party>
    </cac:AccountingCustomerParty>
    <!-- Code omitted for clarity -->
</Invoice>
